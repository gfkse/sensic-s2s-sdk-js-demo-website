This page allows to call the SUI Touchpoint (suitp.html) with parameters as specified in the fields above (pid, hhid, uid, paneluserid, adid).

To have a publisher-like context, and make it possible to check cross-domain behavior, this page should not be called with its sensic.net URL, but with amazonaws.com URL.

The field "SUI from Connector" displays the current SUI, which is fetched using the SUI Connector. This, in any case, happens without calling SUI Generator (what the SUI Connector normally could do), so the current SUI could be null.

When you click on Call Touchpoint by window, the touchpoint page is called with parameters as specified above. Additionally a "go" parameter is passed, which instructs the touchpoint to redirect to this page again.

After the touchpoint was called, we have a touchpoint-generated SUI ID, which is displayed here.

With the redirection, this page calls the SUI Connector again. Therefore it can be tested whether the touchpoint-generated SUI, set in the sensic.net domain, is really active in the current domain.

For example, if sensic.net third-party cookies are blocked (which in Firefox is the case by default), the SUI from the SUI Connector will always be null, although the touchpoint sets a SUI as expected.

When you click on Call Touchpoint by iframe, the touchpoint page is called in similar way, but in an iframe, not in the top-level window. The result of that call is propagated to the top-level window by message. Usually, such an iframe would not be visible. However, here it is shown while in action, so you can see that something is happening.
